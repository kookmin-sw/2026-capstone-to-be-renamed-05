Problem

회계사 채용 정보는 아직 너무 흩어져 있습니다

지원자는 여러 채널을 오가며 공고 조건과 회사 정보를 직접 대조해야 하고, 기업은 지원자의 관심 흐름을 파악하기 어렵습니다.

01

공고의 파편화

03

Engineering

정적 웹, API, 데이터베이스를 분리한 npm workspace 모노레포입니다

GitHub Pages에서 프로젝트를 볼 때 필요한 아키텍처, 워크스페이스 구조, 실행 명령과 검증 명령을 한곳에 정리했습니다.

System Architecture

채용 공고 목록 조회 흐름

사용자는 CloudFront를 통해 S3에 배포된 Next.js 정적 페이지를 받아 보고, API 요청은 고정 접속 주소와 Caddy 라우팅을 거쳐 EC2의 NestJS 서버로 전달됩니다. NestJS는 JobsService와 Prisma ORM을 통해 PostgreSQL의 Jobs, Companies, Sources, Labels 데이터를 조회하고, API 로그는 CloudWatch Logs로 수집합니다. 회사 로고와 이미지 파일은 별도 S3 버킷에서 정적 자산으로 관리합니다.

Workspace

  • apps/web: Next.js App Router 기반 사용자·관리자 UI
  • apps/api: NestJS REST API, Swagger/OpenAPI, 인증과 권한
  • packages/shared: enum, 공통 타입, DTO 친화 모델
  • prisma: schema, migrations, seed/mock 데이터

Core Stack

  • Web: TypeScript, Tailwind CSS, shadcn/ui, lucide-react
  • API: NestJS, Prisma ORM, PostgreSQL
  • Auth: username/password, argon2, HTTP-only JWT cookie
  • Roles: JOB_SEEKER, COMPANY, ADMIN

Product Scope

  • Phase 0는 내부 지원서가 아니라 공고 발견과 큐레이션에 집중
  • 공고 데이터는 출처와 마지막 확인 시간을 함께 노출
  • 기업회원 요청은 관리자 검수 후 게시
  • CSV import, URL preview, AI labels는 untrusted input으로 처리

Repository Checklist

  • npm workspaces로 web, api, shared 패키지를 한 저장소에서 관리
  • Prisma schema와 migration으로 DB 구조를 버전 관리
  • Swagger/OpenAPI로 API 명세를 확인할 수 있도록 구성
  • GitHub 저장소와 서비스 데모 링크를 페이지 상단에서 바로 접근
